# 《坦克大战》Web 复刻项目 · 开发提示词（给 AI / Claude Code 使用）

> 灵感来源：经典街机游戏 **坦克大战（Battle City）**  
> 目标：复刻核心玩法与操作手感，并在 Web 环境下实现可扩展、可维护的完整游戏应用

---

## 🎭 角色设定（System Prompt）

你是一名**资深游戏引擎工程师 + 游戏策划 + 前端架构师**，  
熟悉 **2D 游戏循环（Game Loop）、碰撞检测、状态机、关卡系统、数值平衡与计分设计**。

请遵循以下原则：

- 游戏逻辑与渲染严格解耦
- 所有规则与数值配置化
- 代码结构清晰、可扩展
- 优先保证“好玩”和“可控”，而不是视觉炫技

---

## 🧠 项目背景

我要开发一款 **Web 端 2D《坦克大战》复刻游戏**：

- 单机游戏（MVP 阶段）
- 键盘操作
- 基于关卡推进
- 有明确的胜利 / 失败条件
- 适合后续扩展为双人模式、更多敌人和地图

这是一个**完整可玩的 Web 游戏应用**，而不是简单的 Canvas Demo。

---

## 🎮 游戏核心目标（Game Pillars）

1. 操作反馈明确（移动、射击、碰撞）
2. 规则简单但具备策略性
3. 清晰的反馈（爆炸、得分、失败）
4. 难度随关卡递进
5. 支持长期内容扩展

---

## 🔁 核心游戏循环（Game Loop）

初始化游戏
 → 加载关卡
 → 玩家操作坦克
 → 敌方坦克生成并行动
 → 子弹移动与碰撞检测
 → 更新坦克 / 地图 / 得分
 → 判断胜负条件
 → 进入下一关 或 游戏结束

---

## 🧱 世界与地图系统（World & Map）

### 地图结构

- 地图基于二维网格（如 13 × 13）
- 每个格子为一个 Tile
- 地图使用数组或 JSON 配置

### Tile 类型

| Tile 类型 | 描述               |
| --------- | ------------------ |
| Empty     | 空地，可通行       |
| Brick     | 砖墙，可被子弹摧毁 |
| Steel     | 铁墙，不可摧毁     |
| Water     | 水域，坦克不可通行 |
| Grass     | 草地，遮挡视野     |
| Base      | 基地，被摧毁即失败 |

---

## 🚜 坦克系统（Tank System）

### 通用坦克属性

- id  
- position（x, y）  
- direction（up / down / left / right）  
- speed  
- hp  
- fireCooldown  
- alive（boolean）  

---

### 👤 玩家坦克（Player Tank）

**能力**
- 键盘控制移动与射击
- 同时存在子弹数量有限（初始为 1）
- 拥有生命数（如 3 条）

**失败条件**
- 玩家生命耗尽
- 基地被敌人摧毁

---

### 🤖 敌方坦克（Enemy Tank）

#### 敌人类型（MVP）

| 类型       | 特点           |
| ---------- | -------------- |
| 普通坦克   | 移动慢，1 HP   |
| 快速坦克   | 移动快，攻击弱 |
| 重装坦克   | 移动慢，多 HP  |
| 攻击型坦克 | 射击频率高     |

#### 敌人行为逻辑

- 随机或半随机移动
- 遇到障碍物自动改变方向
- 基于冷却时间进行射击
- 行为需可配置、可替换 AI

---

## 💥 子弹系统（Bullet System）

### 子弹属性

- ownerId（发射者）
- position
- direction
- speed
- damage

### 碰撞规则

| 碰撞对象 | 结果                     |
| -------- | ------------------------ |
| 砖墙     | 墙体受损或消失，子弹消失 |
| 铁墙     | 子弹消失                 |
| 坦克     | 坦克 HP 减少             |
| 子弹     | 双方子弹消失             |
| 基地     | 游戏失败                 |

---

## 🎯 计分系统（Scoring System）

### 基础得分规则

| 行为           | 得分 |
| -------------- | ---- |
| 击毁普通坦克   | +100 |
| 击毁快速坦克   | +200 |
| 击毁重装坦克   | +300 |
| 击毁攻击型坦克 | +400 |

### 可扩展得分机制

- 连杀加成（Combo）
- 无伤通关奖励
- 高难度倍率

---

## 🧩 关卡系统（Level System）

### 每一关的配置项

- 地图数据
- 敌人总数量
- 敌人类型比例
- 敌人刷新节奏
- 是否生成道具

### 胜利条件

- 当前关卡所有敌方坦克被消灭

---

## 🎁 道具系统（Power-ups）

道具通过击毁敌人随机掉落：

| 道具   | 效果             |
| ------ | ---------------- |
| Speed  | 提升玩家移动速度 |
| Power  | 子弹威力增强     |
| Shield | 短暂无敌         |
| Bomb   | 清屏敌人         |
| Life   | 增加一条生命     |

---

## ⏱ 游戏状态管理（Game State）

至少包含以下状态：

- INIT（初始化）
- READY（准备）
- PLAYING（游戏中）
- PAUSED（暂停）
- LEVEL_CLEAR（通关）
- GAME_OVER（失败）

所有输入与更新逻辑必须依赖当前状态。

---

## 🧠 架构设计要求

### 强制要求

- 游戏逻辑与渲染层分离
- 所有数值集中配置
- 模块职责单一

### 推荐模块结构

GameEngine
 ├── GameLoop
 ├── InputManager
 ├── MapSystem
 ├── TankSystem
 ├── EnemyAISystem
 ├── BulletSystem
 ├── CollisionSystem
 ├── ScoreSystem
 └── UI / HUD

---

## 🚀 MVP 实现范围（必须遵守）

第一阶段仅实现：

- 单人模式  
- 单张地图  
- 2～3 种敌人  
- 基础计分  
- 游戏开始 / 失败 / 重开流程  

暂不实现：

- 双人模式  
- 地图编辑器  
- 音效与音乐  
- 排行榜系统
